### 2) Import des librairies et fonctions import pandas as pd import os, globdef load_csv_folder(folder_path): """ Charge tous les fichiers CSV d'un dossier et les concatène dans un seul DataFrame """ files = glob.glob(os.path.join(folder_path, "*.csv")) df_concat = pd.concat([pd.read_csv(file) for file in files], ignore_index=True) return df_concat ### 3) Import des données df_sales = load_csv_folder(DATA_FOLDER) df_products = pd.read_excel(PRODUCT_FILE) print(f"[INFO] Nombre de lignes importées depuis {DATA_FOLDER} : {len(df_sales)}") ### 4) Traitement des données # jointure avec le référentiel produit df_all = pd.merge(df_sales, df_products, on="id produit", how="left", validate = "m:1")# calcul du chiffre d'affaires = quantité × prix unitaire df_all["chiffre d'affaires"] = df_all["quantité"] * df_all["prix unitaire"]# agrégation par catégorie df_summary = ( df_all .groupby("catégorie") .agg({"chiffre d'affaires": "sum", "quantité": "sum"}) .sort_values(by="chiffre d'affaires", ascending=False) )# contrôle de cohérence df_missing = df_all[df_all["catégorie"].isna()] if len(df_missing) > 0: print(f"[ALERTE] Produits sans correspondance détectés, {len(df_missing)} lignes concernées") ### 5) Export des résultats df_summary.to_excel(OUTPUT_FILE, sheet_name="Bilan") print(f"[INFO] Export terminé : {OUTPUT_FILE}")

L'enjeu

Quand vos outils actuels atteignent leurs limites.

Excel s'est imposé comme l'outil analytique par défaut des équipes métiers, y compris pour des usages qu'il n'a jamais été conçu pour supporter : fichiers de plusieurs centaines de milliers de lignes, traitements complexes, automatisations, croisements multi-sources. Les équipes métiers se sont alors tournées vers des solutions de contournement. Chacune avec ses mérites et ses propres limites.

🐌

Excel poussé hors de sa zone

Excel sature lorsqu'on lui demande de gérer des centaines de milliers de lignes ou des traitements complexes en chaîne. Il n'est pas fautif, simplement pas conçu pour.

🔁

Manipulations chronophages

Des tâches répétitives refaites à la main chaque semaine, sources d'erreurs et de temps perdu qui ne se récupère pas.

🔗

Sources multiples à croiser

Rapprocher plusieurs fichiers de formats différents peut s'avérer être un véritable casse-tête là où un script Python offre une grande flexibilité.

🔧

Des alternatives plafonnées

VBA est lent, et Microsoft pousse vers d'autres solutions. Power Query et Power Pivot atteignent rapidement leurs limites sur des workflow complexes, compliquent la maintenabilité, et se limitent au requêtage. Fautes de bonnes pratiques, ces fichiers deviennent souvent illisibles.

💸

Outils no-code coûteux, opaques et limités

Alteryx, Zapier et leurs équivalents reproduisent trop souvent à prix fort ce qu'un script Python ferait gratuitement. Ils sont mal documentés, peuvent provoquer des comportements inattendus, et sont bornés aux fonctionnalités prévues par l'éditeur.

🤷

Scripts vibe-codés non maîtrisés

Certains ont déjà pris le réflexe de demander un script Python à une IA. C'est un excellent point de départ. Mais sans comprendre ce que le script fait, impossible de vérifier qu'il est juste, de corriger une erreur, ou de l'adapter. On reste dépendant d'un outil qu'on ne maîtrise pas.

Chaque outil a rendu service à sa manière. Mais aucun n'apporte une solution complète : donner aux profils métiers analytiques un outil puissant, lisible, maintenable et suffisamment polyvalent pour aller au-delà du seul traitement de données. C'est exactement ce qu'offre Python.

Python et Excel

Python ne remplace pas Excel.
Il fait ce qu'Excel ne peut pas faire.

Excel reste irremplaçable pour ce qu'il fait le mieux : explorer, présenter et partager vos données. C'est là qu'il excelle, et cette formation le valorise en tant que tel.

Python entre en jeu pour tout ce qui précède : nettoyer les sources brutes, croiser des fichiers de formats différents, automatiser les traitements récurrents, gérer les gros volumes. Et il vous rend vos résultats dans un fichier Excel propre, prêt à être partagé.

L'objectif est de mieux travailler avec la donnée. Plus vite, avec plus de fiabilité, et en pleine maîtrise de ce que vous produisez. Vous conservez Excel là où il est indispensable, et Python prend en charge ce qu'Excel ne sait pas faire.

Python Croiser plusieurs fichiers, même volumineux, et de formats différents (CSV, Excel, JSON…)
Python Automatiser des traitements récurrents de façon robuste et maintenable
Python Aller au-delà du traitement de données : mail, PDF, API...
Excel Présenter, modéliser simplement et partager vos résultats avec votre équipe
Excel Explorer, analyser et communiquer autour de la donnée

Python et Excel ne s'opposent pas : ils se complètent. Chacun là où il est le plus efficace.

Le vibe-coding est un excellent point de départ, et cette formation vous apprend d'ailleurs à en tirer encore plus parti. Mais s'y limiter, c'est accepter de dépendre d'un résultat que vous ne pouvez ni vraiment vérifier, ni facilement faire évoluer. L'IA produit souvent une très bonne base mais qui reste perfectible. Sans comprendre le script obtenu, impossible de repérer une erreur, d'apporter les dernières adaptations, ou de savoir si le résultat est fiable avant de le partager à votre équipe. Comprendre Python, c'est ce qui transforme l'IA en véritable accélérateur, plutôt qu'en boîte noire à laquelle vous faites confiance par défaut. En outre, les cas d'usages quotidiens sont généralement plus faciles à traiter par soi-même que par l'intermédiaire d'une IA.
Un tutoriel YouTube vous montre comment faire, souvent orientée pour un public tech, sur un cas générique. Cette formation vous apprend à comprendre ce que vous faites : pourquoi ça fonctionne, comment l'adapter à vos propres données, comment corriger une erreur, comment challenger ce qu'une IA vous produit. La différence se sent dès le premier vrai cas métier que vous devez résoudre seul. L'environnement métier est au cœur de tout : le support pédagogique, les bonnes pratiques, les cas d'usage… Tout est pensé pour quelqu'un qui travaille avec des données d'entreprise, pas pour un futur développeur.
